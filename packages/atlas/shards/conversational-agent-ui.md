# Conversational Agent UI

Conversational agent UI is the surface where a human and an autonomous (or semi-autonomous) agent share a turn-based thread. The thread is the source of truth: every tool call, citation, and approval is anchored to a message, not hidden in a side log. The substrate may be a custom web app (Claude.ai, Cursor chat) or a borrowed messenger (Telegram, Discord, Slack); design defaults below assume custom-web but flag the messenger-substrate deltas.

## Archetypal layout patterns

- Vertical scroll, latest at bottom (chat thread). Scroll anchors to bottom on send; user scroll-up pauses auto-scroll until a "jump to latest" pill is dismissed.
- Asymmetric bubble layout: user right-aligned with a tinted background, agent left-aligned neutral or borderless. Avatars optional on agent side; never on user side in 1:1 chat.
- Sticky composer at bottom — text input, attachment trigger, send button, slash-command picker. Composer height grows with content up to a cap (~40% viewport), then internal-scrolls.
- Status indicators between turns or in-bubble: typing dots, "thinking", "tool-using", "waiting on approval", "queued". Status text reads as a verb in present tense.
- Tool-use cards inline (e.g., "Reading `foo.ts`", "Running `bash deploy.sh`", "Searching web for X"). Collapsed by default; expand reveals args + result preview.
- Citation/source pills attached to claims — small chips at the end of a sentence or paragraph, hover or tap reveals quote + URL.
- Slash-command popover: `/`-trigger over input opens a fuzzy-filtered menu of commands with descriptions and argument hints. Esc / click-out dismisses.
- File attachment chips with previews — image thumbnail, file-type icon for non-image, removable with x. Drag-drop into thread or composer.
- Side panel for long artifacts (code, document, image generated by agent) — chat stays narrow (~640-720px) on the left; panel takes remaining width and persists across turns until dismissed.
- Approval prompts inline (not modal). Approval card shows action summary, expanded args (collapsible), Approve / Deny / Modify buttons. Result of approval renders as next agent turn.
- Streaming token render: characters appear progressively with a visible caret/cursor at the head of the streaming text. Mid-stream the bubble grows without re-flowing prior turns.
- Persistent system prompt / context indicator (lightweight) — small header chip "Mike Lawdbot · Haiku 4.5 · workspace loaded" gives the user grounding without a settings page.

## Default DNA — token defaults

- **User bubble**: subtle tinted background — `#EFF6FF` light / `#172554` dark (primary-50 / primary-950 equivalent). Never high-contrast brand color; the user's words are not a CTA.
- **Agent bubble**: neutral or borderless — `#FFFFFF` light / `#0A0A0A` dark, with a hairline border `#E5E7EB` light / `#27272A` dark optional for separation. Code snippets inside agent bubbles get a mono font and a darker inset bg `#FAFAFA` light / `#18181B` dark.
- **Typography**: sans body 15px desktop / 16px mobile (line-height 1.55), bumped to 16-17px when the substrate is a messenger app's web view to match thumb-zone reading. Mono for code at 13-14px (line-height 1.5). Font stack: Inter / SF Pro / system-ui body, JetBrains Mono / SF Mono / Menlo code.
- **Spacing**: bubble inner padding 12px y / 16px x. Between turns 16px (same speaker) / 24px (speaker change). Composer outer margin 16px on mobile, 24px on desktop.
- **Radii (`rounded`)**: bubbles 16px (large but not pill). Composer 12px. Inline tool-use cards 8px. Attachment chips 8px. Buttons 6-8px.
- **Motion**: typing-dot animation (3 dots, 1.4s ease loop, opacity 0.3→1.0). Fade-in 120ms for new turns. Streaming text appears with no jank — never re-layout sibling turns. `prefers-reduced-motion: reduce` replaces typing dots with static `…` and disables fade-in.
- **Status colors** (border or icon tint, not full bubble fill):
  - thinking: `#94A3B8` (slate-400)
  - tool-using: `#0EA5E9` (sky-500)
  - waiting on approval: `#F59E0B` (amber-500)
  - error: `#F43F5E` (rose-500)
  - success: `#10B981` (emerald-500)
- **Code-block tokens**: dark-mode default for code (`#0A0A0A` bg, `#E4E4E7` fg) regardless of overall theme — code reads better dark. Light-mode code is acceptable but lower-priority. Syntax theme: One Dark Pro or shiki `github-dark`.
- **Touch targets** (messenger-substrate): minimum 44x44px for send/attach/slash buttons.

## Exemplars

- **Claude.ai** — artifacts side panel, slash commands, citation pills, dark-default code blocks. Reference for separating long artifacts from the chat stream.
- **ChatGPT** — canvas (parallel to artifacts), custom GPTs, model badge in turn headers. Reference for status indicators ("Thought for 8s") as collapsed transcript.
- **Cursor chat panel** — composer with file-mention `@`-trigger alongside `/`-trigger, inline diff approvals. Reference for tight integration of approval UI with code edits.
- **Linear Asks** — formal but conversational; threading inside a workspace primitive (issue) rather than a standalone chat. Reference for chat-as-collaboration-mode rather than chat-as-app.
- **Telegram** — the substrate Mike Lawdbot uses. Bubble defaults baked in by the platform; design space is bot replies (formatting, inline keyboards for approvals, attachments). Reference for messenger-substrate constraints (no custom side panel, must use inline-keyboard buttons for approval prompts).
- **Discord** — slash commands as the dominant UX language across an ecosystem; teaches users that `/` is the affordance for power.
- **Anthropic Workbench** — system prompt + message history side-by-side; reference for split-view chat where the inputs and outputs are equally first-class.

## Anti-patterns

- Modal dialogs for approval — interrupts flow, breaks scroll context. Use inline approval cards anchored to the requesting agent turn.
- Hidden tool-use — never silently execute tools. Always surface what the agent did, even if collapsed by default. Trust requires legibility.
- Bubble-style UI without distinguishable user/agent treatment — if the only differentiator is alignment, scan-fail at thread length > 10 turns. Use bg tint + alignment together.
- Centered chat alignment — kills scan order; humans read in F-pattern down the left edge.
- Auto-collapsing the latest message — the freshest turn must be fully visible without interaction.
- Streaming without a visible cursor or progress signal — looks frozen on slow networks; the streaming-cursor is the heartbeat.
- Send-on-Enter with no Shift+Enter for newlines (or vice versa) — be explicit. Document the binding in a hint near the composer (`Enter to send · Shift+Enter for newline`).
- Loading spinners with no text — "thinking…" / "calling tool: web_search…" beats a bare spinner. Empathy beats animation.
- Persistent toasts that obscure the composer — toasts must clear within 4s or relocate above the composer, never on top of it.
- Bubble-color = brand-color on the user side — turns the user's own words into a CTA, breaks visual hierarchy.
- Auto-scroll override during user scroll-up — respect the read intent; show a "jump to latest" pill instead.
- Approval buttons that don't preview the action — "Approve" with no expanded args is a footgun. Always show what is being approved.

## Fingerprint signals

Observable cues a project is in this category, used by `pipeline-phases.md` Step 0.5 for atlas fallback routing.

- **Files**:
  - Streaming chat handlers — `route.ts` / `app/api/chat/route.ts` exporting `POST` with `streamText`, SSE handlers, or WebSocket upgrade
  - Slash-command registries — files named `slash-commands.{ts,tsx}`, `commands.json`, or `commands/*.{ts,md}`
  - Tool-call schemas — `tools.ts` with Zod schemas, or `tools/*.ts` exporting `{name, description, parameters, execute}`
  - Workspace / persona files — `SOUL.md`, `USER.md`, `AGENTS.md` in a `workspace/` or `.openclaw/` dir (Mike-Lawdbot/OpenClaw fingerprint)
  - Heartbeat / cron entrypoints — `heartbeat_*.py`, GitHub Actions `heartbeat.yml`, or scheduled webhook endpoints (Gravity Claw fingerprint)
- **Dependencies** (`package.json` / `pyproject.toml`):
  - `ai` (Vercel AI SDK), `@ai-sdk/anthropic`, `@ai-sdk/openai`
  - `@anthropic-ai/sdk`, `openai`, `litellm` (Python)
  - `socket.io`, `socket.io-client`, `ws`, or SSE polyfills
  - `react-markdown`, `remark-gfm`, `rehype-highlight`
  - `shiki` or `prismjs` (code rendering)
  - `node-telegram-bot-api`, `python-telegram-bot`, `telegraf` — Telegram substrate
  - `discord.js` — Discord substrate
  - `google-adk`, `qdrant-client` — agent-graph patterns (Gravity Claw)
- **Patterns**:
  - `useChat` from `ai/react` or `@ai-sdk/react` (Vercel AI SDK)
  - `streamText({ model, messages, tools })` server-side
  - `messages: { role: 'user' | 'assistant' | 'tool', content }` arrays
  - Role-based rendering — `messages.map(m => m.role === 'user' ? <UserBubble/> : <AgentBubble/>)`
  - Tool-call rendering — checking `m.toolInvocations` or `m.parts` and rendering tool-use cards
  - Webhook handlers for `/webhook` POST (Telegram/Discord/Slack inbound)
  - Whitelist / allowlist checks on inbound message user IDs (messenger-substrate)
- **Anti-fingerprint** (suggests _not_ this category):
  - Heavy data tables, sortable columns → SaaS dashboard
  - Hero + CTA + photo grid → marketing/landing
  - Real-time price tickers, sparkline cells → trading/analytics
  - Two LLMs side-by-side responding to one prompt → multi-LLM synthesis (cousin category — share streaming + bubbles, but layout is split-pane not single-thread)
